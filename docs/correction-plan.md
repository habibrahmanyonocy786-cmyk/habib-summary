## ۱. مشکلات امنیتی (High Priority)

### 1.1. XSS - عدم Sanitize در نمایش مقالات

**فایل:** `assets/js/app.js:291`  
**توضیح:** عنوان مقاله (`a.title`) و خلاصه مستقیماً در `innerHTML` قرار می‌گیرند بدون هیچ sanitization. اگر داده JSON حاوی کد HTML مخرب باشد، XSS رخ می‌دهد.  
**راه‌حل:** استفاده از `textContent` بجای `innerHTML` یا عبور دادن داده از Sanitizer موجود در `src/utils/sanitize-html.js`

```javascript
// قبل:
innerHTML = `<h4>${a.title}</h4>`
// بعد:
innerHTML = `<h4>${Sanitizer.sanitize(a.title)}</h4>`
```

### 1.2. افشای Credentials ادمین

**فایل:** `admin/login.html:41`  
**توضیح:** نام کاربری و رمز عبور به صورت Plain Text در کد کلاینت ذخیره شده است:
```javascript
if (u === 'admin' && p === 'habib2026')
```
**راه‌حل:** 
- حذف احراز هویت سمت کلاینت
- پیاده‌سازی احراز هویت سمت سرور با JWT
- استفاده از hash (bcrypt) برای رمز عبور
- یا حداقل استفاده از localStorage/環境変数

## ۲. مشکلات معماری (Medium Priority)

### 2.1. از دست رفتن فیلتر مقالات با تغییر Tab

**فایل:** `assets/js/app.js:301`  
**توضیح:** تابع `filterArticles` مستقیماً DOM را تغییر می‌دهد و فیلتر در state ذخیره نمی‌شود. با تغییر Tab و برگشت، فیلتر ریست می‌شود.  
**راه‌حل:** ذخیره `selectedCategory` در `Habib.state` و اعمال مجدد فیلتر هنگام `renderArticles()`

### 2.2. کد تکراری استخراج متن

**فایل:** `assets/js/app.js:406-423`  
**توضیح:** کد استخراج متن از PDF/DOCX/Image در `app.js` کپی شده در حالی که ماژول‌های مجزا وجود دارند:
- `assets/js/extract-text.js` - TextExtractor
- `assets/js/ocr.js` - OCR
- `assets/js/viewer-pdf.js` - PDFViewer
- `assets/js/viewer-docx.js` - DOCXViewer

**راه‌حل:** استفاده از `TextExtractor` از `extract-text.js` بجای کد تکراری در `app.js`

### 2.3. Utility files استفاده نشده

**فایل:** `src/utils/storage.js` و `src/utils/sanitize-html.js`  
**توضیح:** این فایل‌ها وجود دارند اما در `app.js` یا هیچ جای دیگر استفاده نمی‌شوند.  
**راه‌حل:** 
- ادغام `sanitize-html.js` در pipeline نمایش مقالات و خلاصه‌ها
- استفاده از `storage.js` برای ذخیره نتایج آزمون‌ها و تنظیمات کاربر

### 2.4. URLهای جعلی مقالات

**فایل:** `data.json`  
**توضیح:** تمام URLهای مقالات به دامنه اصلی (مثلاً `https://www.bbc.com`) اشاره دارند نه به مقاله واقعی.  
**راه‌حل:** جایگزینی با URLهای واقعی یا حذف لینک "مشاهده منبع"

### 2.5. استایل نادرست دکمه فیلتر فعال

**فایل:** `assets/js/app.js:306`  
**توضیح:** دکمه فیلتر فعال کلاس `tag tag-negative` (قرمز) می‌گیرد بجای کلاس مجزا برای حالت active.  
**راه‌حل:** افزودن کلاس `tag-active` با استایل مجزا (مثلاً آبی تیره)

## ۳. مشکلات UI/UX (Low Priority)

### 3.1. نوار پیشرفت سوالات

**فایل:** `assets/js/app.js:160`  
**توضیح:** نوار پیشرفت در سوال اول ۰٪ را نشان می‌دهد که گیج‌کننده است:  
```javascript
const pct = (this.state.qIdx / total) * 100; // 0/3 = 0%
```
**راه‌حل:** استفاده از `((this.state.qIdx) / total) * 100` برای نمایش پیشرفت واقعی

### 3.2. نبود وضعیت Loading

**فایل:** `assets/js/app.js:272`  
**توضیح:** هنگام بارگذاری مقالات فقط متن ساده "در حال بارگذاری مقالات..." نمایش داده می‌شود.  
**راه‌حل:** افزودن Skeleton loader یا Spinner مشابه بخش آپلود

### 3.3. عدم دکمه بازگشت در بخش سوالات

**فایل:** `assets/js/app.js:152`  
**توضیح:** بعد از انتخاب یک کتاب برای آزمون، راهی برای بازگشت به لیست کتاب‌ها نیست (بجز تغییر Tab).  
**راه‌حل:** افزودن دکمه "بازگشت به لیست کتاب‌ها" در بالای بخش سوالات

### 3.4. Reset آزمون ترکیبی

**فایل:** `assets/js/app.js:251`  
**توضیح:** دکمه "شروع مجدد" همه state را پاک می‌کند اما اگر کاربر صفحه را Refresh کند، state از دست می‌رود.  
**راه‌حل:** ذخیره state آزمون در `sessionStorage` یا استفاده از `storage.js`

### 3.5. نبود Meta Tags برای SEO

**فایل:** `index.html`  
**توضیح:** تگ‌های Open Graph (og:title, og:description, og:image) وجود ندارند.  
**راه‌حل:** افزودن تگ‌های OG و Twitter Card

### 3.6. نبود Favicon

**فایل:** `index.html`  
**توضیح:** هیچ Favicon برای سایت تعریف نشده است.  
**راه‌حل:** افزودن favicon با فرمت‌های مختلف (ico, png)

### 3.7. کامپوننت‌های HTML استفاده نشده

**فایل:** `src/components/`  
**توضیح:** فایل‌های `header.html`, `footer.html`, `nav-tabs.html` به صورت static در `index.html` کپی شده‌اند.  
**راه‌حل:** بارگذاری داینامیک این کامپوننت‌ها با JavaScript Fetch

## ۵. بهبودهای پیشنهادی (Feature Requests)

علاوه بر رفع مشکلات، بهبودهای زیر برای نسخه‌های بعدی پیشنهاد می‌شود:

7. **آمار پیشرفت** - نمودار پیشرفت در آزمون‌ها
8. **دانلود خلاصه** - امکان دانلود خلاصه به صورت PDF یا TXT
